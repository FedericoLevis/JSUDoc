1) JSU Validate API

Very simple but powerful Validate APIs: cValidate() take care of both Validation and Presentation:
  • Validation: All the constraints described in Validate.js v. 0.10.0 are supported by in the first constraints parameter of cValidate().
  • Presentation:
    • You can choose many options to display Validate Errors in single Red Items, Cumulative Section, Popup,...
      No additional code is required: cValidate take care of everything
    • The cValidate() constraints JSON parameter supports also the optional tip field, used to setup a JSU Floating Tooltip on the right of the item to validate

As indicated before, JSU validate API implements all the constraints of the Open Source Component Validate.js v. 0.10.0 , very well described in Validate.js:

Follow JSU Validate APIs:
Validate APIs & CSS
JSU File JSU API Item Description
jsu/core/JQPopup/Popup.js cValidate() constructor Create cValidate class that manage both item validation and presentation of Validation Errors
cValidate.validateApply() Method Apply Validation of all the Items and display Validation Errors (e.g in Section or Popup)
For example you can call this method in the onclick of "Submit" Button
cValidate.setOption() Method Change run-time the Validate Option, that were set with cValidate() constructor
jsu/core/jsu.css JSU Validate CSS Validate CSS: Color, Background, Font, Border...

1.1) DEFAULT Validate OPTIONS

DEFAULT Validate OPTIONS
SAMPLES DESCRIPTION
  • With default Validate Options, Items are instantly validated at onchange event
  • Clicking Submit button, all the items will be validated
  • You can Try other Validate Options with the OPTION Button
supports all the constraints of Open Source Component Validate.js v. 0.10.0 [MIT License]
The constraints are passed to JSU API cValidate() in the constraints JSON parameter, that in addition supports also the optional tip field, used to setup a JSU Floating Tooltip on the right of the item to validate.
With JSU Validate API only 2 calls are enought for BOTH constraints validation and UI Presentation (errors, Tips,..):
  • Setup everything (Constraints Validation, Error Presentation, Tooltips) with new cValidate()
  • In a button, or where you prefer, you can call cValidate.validateApply() to Validate all the section
of Sample_1:

cValidate constraints used in this Sample:
  • Email: Mandatory email
  • Duration: Mandatory duration in range [1..10] - Tooltip present
  • Birthday: Optional, if present it must be a date in format YYYY-MM-DD

1.2) Validate OPTION to Show Section with ERRORS

Validate OPTION to Show Section with ERRORS
SAMPLES DESCRIPTION
  • Validate Options bOnErrShowSect=true is used to show a Section with all Validate Errors
  • Click Submit button to see the Error Section
  • You can Try other Validate Options with the OPTION Button
of Sample_2:

cValidate Options of This SAMPLE:
  • Validate all items with cValidate.validateApply() called when Submit Button is clicked
  • Error Section with all the Validate errors displayed on the Top
  • Alarm Gif in the Items with Validation error

cValidate constraints used in this Sample:
    • Username: Mandatory username with constraints explained in the tip
    • Password: Mandatory password with constraints explained in the tip
    • Confirm Password: Mandatory, must be equal to password

Validate OPTION to Show Popup with ERRORS

  • Click Submit button to see the Popup with Validate Errors
  • You can Try other Validate Options with the OPTION Button
of Sample_3:

cValidate Options of This SAMPLE:

cValidate constraints used in this Sample:
